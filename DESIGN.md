# UI Design Notes

这份文档用于降低后续 UI 修改成本。它不替代代码注释，主要记录容易反复踩坑的视觉约束、响应式规则和验证方式。

## 文件分工

- `src/ui.js`：生成主插件面板、弹窗 DOM、按钮文案和净化模式列表渲染。
- `src/events.js`：绑定按钮点击、导入预设、深度清理、弹窗开关等交互。
- `src/core.js`：净化执行逻辑，包括 `scopeTagMode` 对正文处理范围的影响。
- `src/utils.js`：净化模式标签输入解析、内置标签合并、标签展示格式化。
- `style.css`：所有主题、弹窗、主面板、移动端和平板端适配。

## 命名现状

用户可见文案现在使用“净化模式”。代码里仍保留 `scopeTag`、`bl-scope-*` 等历史命名，用于表示一组参与净化范围判断的包裹标签。

后续只改文案时，优先改 `src/ui.js` 中的显示文字；不要为了文案重命名数据字段，否则会影响旧设置迁移。

## 主题规则

- 新增 UI 颜色优先使用 `style.css` 顶部的 `--bl-*` 变量。
- 插件需要跟随 SillyTavern Smart Theme，优先引用 `--SmartTheme*`，再落到固定 fallback。
- 不要在组件局部直接写大片固定色；危险、成功、强调色也应走 `--bl-danger-color`、`--bl-success-color`、`--bl-accent-color` 等变量。
- 主题入口使用下拉菜单，不使用单纯循环按钮。选项固定为“跟随酒馆”、“白色主题”、“暗色主题”。
- 主插件面板、净化模式弹窗、净化模式内的标签编辑/分组管理弹窗、diff/transfer/toast/loading 都需要带上或继承 `data-bl-theme`，避免深浅色主题不一致。

## 弹窗层级

常规插件弹窗保持在 `10000004` 到 `10000006` 区间。导入预设确认弹窗使用更高层级，避免被插件面板或其他弹窗盖住：

- `#bl-preset-import-choice-modal`: `10000008`
- `.bl-toast`: `2147483647`

新增弹窗时先确认它和导入确认、toast 的关系。需要用户立即决策的确认弹窗应该高于普通面板。

## 响应式断点

当前 CSS 主要分三档：

- 手机：`max-width: 600px`
- iOS 平板 / 平板：`601px` 到 `1024px`
- PC：`min-width: 1025px`

平板端的核心目标是“不贴边、不越界、关闭按钮可见”。手机端的核心目标是“减少纵向占用、字号跟随手机页面密度”。PC 端不要默认放大弹窗和字号，尤其是净化模式弹窗。

## 净化模式弹窗

主入口文案是“净化模式”，对应按钮为 `#bl-scope-tags-btn`。

二选一控件为 `.bl-scope-mode-segment`：

- 横向应撑满当前内容区。
- 不要为了撑满而增大字号或纵向高度。
- PC 当前按钮字号约 `14px`，按钮高度约 `42px`。
- 手机当前按钮字号约 `13px`，按钮高度约 `40px` 到 `42px`。

标签列表展示优先级：

1. 备注或用户可读标签名优先显示。
2. 具体 `<tag> ... </tag>` 作为次级信息显示。
3. 不再显示三点省略按钮；省略只作为文本截断效果。
4. 每个分组标题行需要提供独立开关，可一键启用或停用该分组内全部 XML 标签。
5. 分组展开后，标签行之间使用从文本列开始的缩进分割线，避免复选框下方也出现横线。

标签输入提示和校验需要保持一致：允许填写普通标签名、数字开头标签名、完整起始标签，例如 `UpdateVariable`、`<horae>`。

## 修改检查清单

改 UI 后至少检查：

- `node --check src\ui.js`
- `git diff --check`
- PC 视口：`1280x720`
- 手机视口：约 `390x844`
- 平板视口：约 `820x1180` 或 `1024x768`

检查重点：

- 弹窗是否被其他插件面板盖住。
- 关闭按钮是否可见。
- 主面板是否横向溢出。
- 文字是否溢出按钮或遮挡下一行。
- `data-bl-theme="light"`、`dark`、`auto` 下颜色是否仍可读。
